# 🛶 富山湾 出艇判断チェック

富山湾でのカヤックフィッシングで、その日「出る/出ない」を**ひとめで**判断するための簡易Webツールです。
**GO（出られそう）／ CAUTION（要確認）／ STOP（やめる）** の3段階で、今日・明日の判定と根拠の数値を表示します。

> ⚠️ これは簡易的な目安です。最終判断は、海快晴等の専門情報や現地での目視確認と合わせて、必ず**自己責任**で行ってください。

## 判定ロジック（初心者向け）

| 判定 | 条件 |
|------|------|
| **GO** | 風速 3m/s 以下 **かつ** 波高 0.5m 未満 **かつ** オフショア（陸→海）でない |
| **STOP** | 風速 5m/s 以上 **または** 波高 0.5m 以上 **または** オフショアの風 **または** 雷予報あり |
| **CAUTION** | 上記の中間（現地で要確認） |

- 判定は **日中（日の出〜日没）の最も悪い条件**（安全側）で行います。
- **オフショア（沖出し）**：富山湾は北向きに開く湾のため、陸（南側）から海へ吹く**南寄り（南東〜南西）の風**を沖出しと判定します（風速2m/s以上のとき）。
- **予報の信頼度**：気象モデル2系統（JMA系=`jma_seamless` と GFS系=`gfs_seamless`）の風速を比較し、日中の**平均差**が `2.5 + 0.5×何日先` m/s を超えた場合に「予報が割れている＝信頼度が低い」として判定を一段厳しく（GO→CAUTION）します。先の予報ほど自然に差が開くため許容を広げ、**本当にモデルが食い違う時だけ**警告が出るようにしています（しきい値は `DIV_BASE` / `DIV_PER_DAY` で調整可能）。
- **雷**：いずれかのモデルが雷（WMOコード 95/96/99）を予報した場合は**無条件でSTOP**。CAPE（大気不安定の指標）も補助表示します。

## データ出典

- **気象・海象**（風速・風向・波高・うねり・日の出日没・天気・CAPE）：[Open-Meteo.com](https://open-meteo.com/) — [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/)
  - 天気予報 API: `api.open-meteo.com`
  - 海洋（波）API: `marine-api.open-meteo.com`
- **潮汐**（満潮・干潮の時刻と高さ）：[tide736.net](https://tide736.net/)（気象庁 潮位表をもとに算出。富山湾・阿尾＝氷見）

すべてAPIキー不要・無料。ブラウザから直接取得（全APIがCORS対応）するため、サーバー不要の静的サイトとして動作します。

## 設定（地点・しきい値）

`index.html` 冒頭の `//=== 設定 ===` ブロックで変更できます。

```js
const LAT = 36.80, LON = 137.20;   // 富山湾 代表地点
const TIDE_PC = 16, TIDE_HC = 1;   // 潮汐：富山県・阿尾（氷見）
const WIND_GO = 3.0, WIND_STOP = 5.0;  // 風速しきい値 (m/s)
const WAVE_STOP = 0.5;                  // 波高しきい値 (m)
const OFFSHORE_FROM = [135, 225];       // オフショアとみなす風向（吹いてくる方位）
```

潮汐の港コード（`pc`=都道府県, `hc`=港）は [tide736.net](https://tide736.net/) のコード表で確認できます。

## スマホのホーム画面に追加

PWA対応（`manifest.webmanifest`＋アイコン）済み。スマホのブラウザで開き「ホーム画面に追加」するとアプリのように単独表示で使えます。

## GitHub Pages での公開

1. このフォルダ（`index.html`, `manifest.webmanifest`, `icon-*.png`）をGitHubリポジトリにpush
2. リポジトリの **Settings → Pages** で、Branch を `main`（フォルダ `/root`）に設定
3. 数分後に `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます

## ローカルで動作確認

cd kayak-go-check
