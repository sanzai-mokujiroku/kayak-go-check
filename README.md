# 🛶 カヤック出艇判断チェック

カヤックフィッシングで、その日「出る/出ない」を**ひとめで**判断するための簡易Webツールです。
スマホのGPSで**現在地を自動取得**し、その場所の**時間別**（6〜16時／2時間刻み）に
**GO（出られそう）／ CAUTION（要確認）／ STOP（やめる）** を表示します。今日・明日・明後日を切り替え可能。

> ⚠️ これは個人が自分のために作った簡易ツールです。精度の保証はできません。最終判断は、海快晴等の専門情報・現地での目視確認と合わせて、必ず**自己責任**で行ってください。

## 主な機能

- **現在地で判定**：ブラウザの Geolocation API で現在地を取得し、その座標で気象・海象を取得。
- **最寄りの潮汐観測点を自動選択**：全国 712 か所（tide736.net＝気象庁 潮位表）の中から、現在地に最も近い観測点を選びます。
- **時間別表示（メイン）**：6〜16時を2時間刻みで「風速・波高・風向・雷」と各時刻の判定を一覧。
- **総合判断**：その日の全時間帯で**一番悪い**判定を採用（安全側）。
- **撤退の目安時刻**：条件が悪化し始める時刻を表示（例：「※10時以降は条件が悪化します」）。
- **雷**：雷予報のある時間帯を明示し、その時間は STOP。
- **日付切替**：今日・明日・明後日の3日間（デフォルトは今日）。

## 判定ロジック（初心者向け）

| 判定 | 条件（各時刻ごと） |
|------|------|
| **GO** | 風速 3m/s 以下 **かつ** 波高 0.5m 未満 |
| **STOP** | 風速 5m/s 以上 **または** 波高 0.5m 以上 **または** 雷予報あり |
| **CAUTION** | 上記の中間（現地で要確認） |

- **総合判断**は、表示時間帯（6〜16時）の中で**最も悪い時刻**に合わせます。
- **予報の信頼度**：気象モデル2系統（JMA系=`jma_seamless` と GFS系=`gfs_seamless`）の風速を比較し、日中の**平均差**が `2.5 + 0.5×何日先` m/s を超えた場合に「予報が割れている＝信頼度が低い」として判定を一段厳しく（GO→CAUTION）します。
- **雷**：いずれかのモデルが雷（WMOコード 95/96/99）を予報した時間帯は**無条件でSTOP**。

> ℹ️ 旧版（富山湾固定）にあった「オフショア（沖出し）判定」は、海岸の向きが地点ごとに異なり全国では一般化できないため、全国対応にあたり判定条件から外しています。風向は参考表示します。

## データ出典

- **気象・海象**（風速・風向・波高・天気）：[Open-Meteo.com](https://open-meteo.com/) — [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/)
  - 天気予報 API: `api.open-meteo.com`
  - 海洋（波）API: `marine-api.open-meteo.com`
- **潮汐**（満潮・干潮の時刻と高さ）：[tide736.net](https://tide736.net/)（気象庁 潮位表をもとに算出）

すべてAPIキー不要・無料。ブラウザから直接取得（全APIがCORS対応）するため、サーバー不要の静的サイトとして動作します。

## 設定（しきい値・観測点リスト）

`index.html` 冒頭の `//=== 設定 ===` ブロックで変更できます。

```js
const HOURS = [6,8,10,12,14,16];        // 表示する時間帯（2時間刻み）
const WIND_GO = 3.0, WIND_STOP = 5.0;   // 風速しきい値 (m/s)
const WAVE_STOP = 0.5;                   // 波高しきい値 (m)
const DEFAULT_LOC = {lat:36.80, lon:137.20, name:"富山湾（既定）"};  // 位置情報が使えない時の既定地点
```

潮汐観測点は `TIDE_STATIONS`（`[pc, hc, lat, lon, 名前]` の配列）に全国分を内蔵し、現在地から最寄りを自動選択します。
座標は tide736.net の度分（DDMM）表記を**10進度に変換済み**です。

## スマホのホーム画面に追加

PWA対応（`manifest.webmanifest`＋アイコン）済み。スマホのブラウザで開き「ホーム画面に追加」するとアプリのように単独表示で使えます。
※ 位置情報を使うため、HTTPS（GitHub Pages 等）または `localhost` でのアクセスが必要です。

## GitHub Pages での公開

1. このフォルダ（`index.html`, `manifest.webmanifest`, `icon-*.png`）をGitHubリポジトリにpush
2. リポジトリの **Settings → Pages** で、Branch を `main`（フォルダ `/root`）に設定
3. 数分後に `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます

## ローカルで動作確認

```bash
cd kayak-go-check
python3 -m http.server 8000
# ブラウザで http://localhost:8000 を開く
```
